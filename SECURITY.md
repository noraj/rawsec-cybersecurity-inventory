Reporting a Vulnerability
The Rawsec Cybersecurity Inventory takes the security of its data and contributors seriously. If you discover a security vulnerability within this project, please follow the responsible disclosure process below.
What counts as a security vulnerability?

A tool or resource entry linking to a known malicious or compromised domain
Insecure (http://) links that expose users to Man-in-the-Middle attacks
A dependency with a known CVE that affects the build pipeline
A script or workflow that could be exploited to inject malicious content

How to report
Please do not open a public GitHub issue for security vulnerabilities.
Instead, report vulnerabilities privately via one of the following channels:

GitHub Private Vulnerability Reporting — Use the "Report a vulnerability" button under the Security tab of this repository.
Email — Contact the maintainer directly via the contact details on raw.pm.

What to include in your report
Please provide as much of the following as possible:

A description of the vulnerability and its potential impact
The file(s) and line number(s) affected
Steps to reproduce or a proof-of-concept
Your suggested fix (if any)

Response timeline

Acknowledgement: Within 72 hours of receiving the report
Assessment: Within 7 days
Fix / Resolution: Dependent on severity — critical issues will be prioritised

Responsible disclosure
We ask that you:

Give us reasonable time to address the issue before any public disclosure
Do not exploit the vulnerability beyond what is necessary to demonstrate it
Do not access or modify data belonging to other users

We are grateful to researchers who help keep this project and its users safe.
Security Best Practices for Contributors
When contributing to this project, please ensure:

All source and website-url fields use https:// (not http://)
You do not add entries pointing to domains known to be malicious, expired, or parked
You verify that a tool is still actively maintained before adding it
You follow the existing JSON schema exactly to avoid breaking the build pipeline

Thank you for helping keep the Rawsec Cybersecurity Inventory safe and trustworthy.
